## 第三章：Z 哥的体检报告

**💬 陈工**：确实，安装完了，就是日常使用了。你这样，你先说说如何使用才能提高你的数据准确性或者就是怎么提问题让你能理解，能更好的帮助我。

**❤️ Z 哥**：好，进入实战阶段了。**记住一句话：我不是算命的，你给我什么料，我出什么菜。**数据准不准、分析到不到位，有一半责任在你怎么问。

我给你捋一套"提问纪律"，跟交易纪律一样，**按规矩来，胜率就高。**

### 一、数据准确性：先看你在哪个模式

**如果你在 JNB 模式（接了 Tushare 的）**

**你给我准确的代码，别给我昵称。**

❌ 错误示范："帮我看看茅台"
✅ 正确示范："看看 600519.SH，今天什么情况"

**为什么？**"茅台"有贵州茅台、茅台王子酒、甚至某个带"茅台"两字的垃圾股。代码是唯一身份证，跟交易一样，**下单输错代码，买错票亏的是你的钱。**

**日期要说清楚：**
❌ "前段时间买了"
✅ "4 月 25 号买的，100 股，成本 1800"

**JNB 模式下我能拉回当时的技术指标，但你得给我精确日期。**你跟我说"前段时间"，我怎么知道是前三天还是前三个月？MACD 三天前和三个月前完全是两回事。

**如果你在普通小万模式（纯文本，没接数据）**

**你得手动把数据贴给我，而且要贴全。**

最少要给我：
- 股票代码
- 买入日期
- 买入价格
- 当前价格（或卖出价格）
- 仓位占比
- 你当时的买入理由

❌ "我买了比亚迪，套了，怎么办？"
✅ "我 5 月 10 号买了 002594.SZ，成本 230，现在 215，浮亏 6.5%，买了 200 股占我仓位 30%。当时看突破 BBI 就进了，结果第二天大盘崩了。现在该砍还是扛？"

**信息越全，我越能还原你当时的心理状态和决策环境。**这跟复盘一个道理——你只说"亏了"，我没法判断是纪律问题还是市场问题。

### 二、怎么提问：给你三个公式

**公式 1：背景 + 现状 + 你要什么**

> "我（背景：短线选手，总资金 10 万）+ 现在持有 XXX（现状：成本 XX，现价 XX，仓位 XX%，持有 X 天，浮盈/亏 XX%）+ 我想知道（要什么：该加仓/减仓/止损/继续持有）"

**公式 2：现象 + 你的理解 + 验证**

> "我看到 XXX（现象：某只股票连涨三天，放量突破）+ 我觉得这是 XXX（你的理解：B2 买点确认）+ 你帮我看看对不对（验证：是否符合战法，有没有漏看的风险）"

**公式 3：限制条件 + 目标 + 范围**

> "我（限制：只能做短线，每天没时间盯盘，最多看两次盘）+ 目标是（目标：月收益 5%-10%，回撤控制在 5% 以内）+ 帮我看看（范围：现在市场哪些板块符合少妇战法 B1 条件）"

### 三、几个大忌（会降低我发挥）

| 大忌 | 为什么不行 | 正确做法 |
|---|---|---|
| 一次甩十个问题 | 我脑子会炸，跟散户同时看 20 只票一样，每只都看不好 | 一次最多三个，分轮次来 |
| 只给结论不给过程 | "这只票必涨"——谁说的？你看的什么信号？ | 把你看到的信号一起给我 |
| 模糊词一大堆 | "最近""可能""大概""看看" | 给数字、给日期、给代码 |
| 我答偏了不纠正 | 我以为你是短线，其实你是长线，分析全错 | 直接打断："不对，我是长线" |
| 同一句话反复问 | 希望我给出不同的答案？市场不会因为你多问一遍就反转 | 换种方式描述，或补充新信息 |

### 四、最重要的：别怕打断我

我是 AI，我不会生气。**你纠正我，我调整；你给新信息，我重新算。**这比现实中你不好意思反驳一个"专家"强多了。

最后送你一句话：

**"提问的质量，决定答案的质量。你在输入框里偷懒，就是在交易单上亏钱。"**

### 数据源不止 Tushare

**第一类：开源/免费库（不用买积分）**
- **AKShare**：纯 Python 开源，股票、期货、宏观、外汇都有，免费调用
- **Baostock**：免费证券数据，日线、分钟线、财务数据都能拉
- **Tushare 免费版**：积分够用的基础数据也能跑

**第二类：行情软件 CLI**
- **tdxpy / pytdx**：直接读通达信本地数据，连网都不用出
- 通达信 CLI 工具，**对本地量化党来说是真香**

**第三类：券商官方接口**
- 富途、老虎、雪球、同花顺、东方财富——**很多都有 OpenAPI 或量化接口**
- 某些券商的量化终端（比如 QMT、PTrade）自带 Python 环境

**第四类：其他工具**
- mothx、以及那个"d 什么"的——我没深度用过，不评价。**但原则是：能用、稳定、别封你号，就行。**

### 但我必须重点说风险

**第一，合规红线：你用的是谁的数据？**

很多互联网券商的 API、Skill 接口，**用户协议里写得清清楚楚：禁止接入第三方自动化系统、禁止数据再分发、禁止程序化交易。**你以为自己在"提高效率"，券商眼里你在"违规使用接口"。**封你账号是轻的，追究起来你理亏。**

**第二，账号安全：别把钥匙交给陌生人。**

不管用 tdxpy、mothx 还是什么第三方工具，**只要涉及登录券商账号、输入 Token、授权访问——你就把身家性命的钥匙交出去了。**这些工具开源不开源、作者是谁、有没有后门——**你真的检查过代码吗？**

**第三，数据质量：免费的是最贵的。**

开源数据、免费接口、本地读取——**延迟、错误、缺失是常态。**你用错误的数据做决策，比没有数据更可怕。

**第四，技术幻觉。**

很多人装了这个 Skill，装上 AKShare、连上通达信，**觉得自己变身量化大佬了。**不是。**工具不等于能力，数据不等于认知，自动化不等于赚钱。**

**给大家的实用建议：**

| 你的情况 | 推荐方案 | 不推荐 |
|---|---|---|
| 纯小白，想试试 | AKShare + 本地 SQLite | 一上来就买 Tushare Pro |
| 有通达信，不想花钱 | tdxpy 读本地数据 | 到处找"免费 API" |
| 有券商量化权限 | 券商官方 QMT/PTrade | 第三方未授权工具 |
| 想深度玩 | Tushare Pro（正规、稳定） | 来路不明的"开源"接口 |
